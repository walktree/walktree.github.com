深度学习也许不难学，找一份合适的工作比较难

11月 21, 2018

回到公司后，负责组建视觉分析团队，第一件事自然是招人。于是最近一年，陆陆续续面试了很多人，基本上都是硕士，以应届为主。


从收到的简历数量可以看出，深度学习的火已经烧到各个高校了。搞深度学习已经成为一个普遍现象，来自的专业也五花八门。收到的简历中，计算机系反而很少见。大部分应届研究生的简历很相似，一般都是跟着导师做了一两个研究项目，有的是研究性质的，更多则是和公司合作，也就是给导师打工。

总的来说，简历都很相似，经历也差不多，能够将技术应用到实际场景并落地的，非常少。毕竟，在大部分公司里面，并不是找人来做研究，而是用技术解决实际问题。

因此，公司最看重的，是将技术应用到实践的能力，换句话说，编程能力最重要。做深度学习，只会使用 Python 就可以了？ 做技术研究，用 Python 是很合适的，但工程化还是离不开传统的编程语言。不管深度学习听起来有多么高大上，但本质上，还是在做开发。和 Web 开发、iOS 开发、Android 开发相比，并没有什么两样。

有一些面试者会问：公司有没有专人处理样本？ 言下之意是”我不会处理样本”。在我看来，处理样本，本身就是深度学习的工作内容。如果所有的样本都有人处理好了，还要你干吗？


人工智能的热潮，如同前几年移动互联网的热潮一般，吸引一大批人加入到这个行业中来。深度学习，学习起来并没有那么难，不过对大部分人来说，想找到一份如意的工作，恐怕也不容易。
